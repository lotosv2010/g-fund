# Code Review 标准

> 完整评分量规见 `.claude/skills/feat/references/review-rubric.md`。

## 六维评分（满分 100）

| 维度 | 权重 | 核心关注点 |
|------|------|-----------|
| 正确性 | 25 | 逻辑缺陷、边界处理、异常传播、幂等性 |
| 可读性 | 20 | 命名自解释、控制流扁平、函数 ≤30 行 |
| 可维护性 | 20 | 单一职责、依赖方向、无重复代码 |
| 安全性 | 15 | 输入校验、密钥管理、SQL 注入防护 |
| 性能 | 10 | N+1 查询、并行化、MCP 并发调用 |
| 测试覆盖 | 10 | AC 覆盖完整性、边界用例、测试在 `tests/` |

评级：≥90 ✅ / 75-89 🟡 / 60-74 🟠 / <60 🔴

## 严重度

| 级别 | 含义 | 阻塞合并 |
|------|------|---------|
| 🔴 Critical | Bug / 安全漏洞 / 架构红线违反 | 是 |
| 🟠 Major | 影响可维护性 / 性能 | 否，强烈建议 |
| 🟡 Minor | 代码质量可改进 | 否 |
| 🔵 Nitpick | 风格偏好 | 否 |

**速查 Critical**：`any` / 空 catch / apps 间直接 import / 硬编码密钥 / SQL 注入

## 本项目检查重点

- 仓位计算：买入/卖出在单个 DB 事务中完成
- MCP 调用：`Promise.allSettled` 并发，单个失败不中断
- SSE 连接：`finally` 块关闭，避免泄漏
- 参数化查询：禁止字符串拼接 SQL（`$1, $2`）
- 测试文件位于 `tests/`，不散落 `src/`
- 集成测试使用真实 PostgreSQL，禁止 mock 数据库
