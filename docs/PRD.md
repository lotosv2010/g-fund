# 产品背景

## 定位

面向个人投资者的基金仓位管理工具，解决以下痛点：
- 持仓分散在多个平台，难以统一查看盈亏
- 缺乏系统性的操作记录习惯
- 不知道自己仓位是否过于集中或存在风险

## 用户画像

- 有一定投资经验的个人基金投资者
- 持有 3-10 支基金，以主动权益类为主
- 希望通过 AI 辅助做出更理性的调仓决策

## 核心价值主张

1. **一站式持仓视图**：统一展示所有基金的成本、净值、盈亏
2. **操作日志习惯化**：每天记录买卖和市场观察，形成投资日记
3. **AI 健康度分析**：定期分析仓位集中度、风险暴露，给出调仓建议

## 导航结构

### 菜单规划

```
┌─────────────┐
│ 📊 总览       ← Dashboard，默认着陆页
│ 💰 基金列表    ← 基金目录 CRUD
│ 📋 交易与持仓   ← 合并页面（持仓 + 交易 + 日志）
│              │
│ ──────────── │
│ 🤖 AI 分析    ← 右侧抽屉，全局可唤起
└─────────────┘
```

### 页面职责

| 页面 | 路由 | 职责 |
|------|------|------|
| 总览 | `/dashboard` | 资产聚合、盈亏曲线、持仓分布、最近操作、快捷入口 |
| 基金列表 | `/funds` | 基金自选库管理（CRUD） |
| 交易与持仓 | `/positions` | Tab 1: 当前持仓 / Tab 2: 买入卖出 / Tab 3: 操作日志 |
| AI 分析 | 抽屉（非路由） | Chat 对话形式，右侧 Drawer 唤起，支持 SSE 流式输出 |

### AI 分析抽屉触发方式

- Dashboard 右下角悬浮按钮
- 各页面 Header 的 AI 图标按钮
- 快捷键 `Cmd+K` 全局唤起

### 设计决策

- **持仓管理与每日日志合并**：交易产生日志，两者天然耦合，合并后实现"看到持仓 → 立即交易 → 查看记录"的操作闭环
- **AI 分析改为抽屉**：避免全页切换打断操作流，Chat 形式更符合 AI 交互习惯
- **新增 Dashboard**：提供全局资产视图，作为用户进入系统的第一站
