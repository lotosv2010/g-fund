# UI 规范

## 设计原则

- 数据优先，减少装饰性元素
- 盈利绿色（`#16a34a`）、亏损红色（`#dc2626`）
- 移动端优先，最小支持 375px 宽度

## 颜色 Token

| Token | 值 | 用途 |
|-------|----|------|
| profit | `#16a34a` | 盈利金额/百分比 |
| loss | `#dc2626` | 亏损金额/百分比 |
| neutral | `#6b7280` | 持平/零值 |
| primary | `#2563eb` | 主操作按钮 |
| surface | `#f9fafb` | 卡片背景 |

## 页面布局

- 侧边导航（桌面）/ 底部导航（移动）
- 顶部 Header：系统名称 + 总盈亏概览

## 组件规范

**PositionTable**
- 列：基金名称、成本净值、当前净值、持有份额、市值、盈亏（金额 + %）、权重
- 盈亏列按正负值使用 profit/loss 颜色
- 底部汇总行：总投入、总市值、总盈亏

**TransactionForm**
- 买入：基金代码（自动补全名称）、金额、净值、日期、备注
- 卖出：基金代码（下拉选已持仓）、份额、净值、日期、备注

**AnalysisReport**
- 风险概览卡片：集中度评分、分散度评分、回撤风险等级
- 调仓建议：Markdown 渲染，支持加粗/列表
- 进度步骤条：fetch_data → risk_analysis → advice → complete
