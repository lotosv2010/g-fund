# UI 规范

## 设计原则

- 数据优先，减少装饰性元素
- 盈利绿色（`#16a34a`）、亏损红色（`#dc2626`）
- 移动端优先，最小支持 375px 宽度

## 颜色 Token

| Token | 值 | 用途 |
|-------|----|------|
| profit | `#16a34a` | 盈利金额/百分比 |
| loss | `#dc2626` | 亏损金额/百分比 |
| neutral | `#6b7280` | 持平/零值 |
| primary | `#2563eb` | 主操作按钮 |
| surface | `#f9fafb` | 卡片背景 |

## 页面布局

- 侧边导航（桌面）/ 底部导航（移动）
- 顶部 Header：系统名称 + 总盈亏概览
- AI 分析以右侧 Drawer 形式全局可用，不占据菜单路由

## 组件规范

### Dashboard（总览页）

**布局**：2 列栅格（桌面）/ 单列堆叠（移动）

| 区域 | 组件 | 内容 |
|------|------|------|
| 顶部统计 | StatCards | 总资产、总盈亏、今日盈亏、持仓数量（4 卡片横排） |
| 左侧主区 | PnLChart | 近 30/90/180 天盈亏曲线（Area Chart，基于 daily_snapshots） |
| 右侧副区 | PositionPie | 持仓分布饼图（按基金类型或风险等级） |
| 底部 | RecentTrades | 最近 5 条交易记录（简洁表格） |
| 右下角 | AiFab | AI 分析悬浮按钮，点击打开 ChatDrawer |

**StatCards 卡片规范**：
- 数值使用 `profit`/`loss` 颜色区分正负
- 百分比变化显示绿色上箭头 / 红色下箭头
- 加载态使用 Skeleton 占位

### PositionTable（持仓表格）

- 列：基金名称、成本净值、当前净值、持有份额、市值、盈亏（金额 + %）、权重
- 盈亏列按正负值使用 profit/loss 颜色
- 底部汇总行：总投入、总市值、总盈亏

### TransactionForm（交易表单）

- 买入：基金代码（自动补全名称）、金额、净值、日期、备注
- 卖出：基金代码（下拉选已持仓）、份额、净值、日期、备注

### TradeLog（操作日志）

- 表格形式，列：日期、基金名称、类型（买入/卖出标签）、金额、净值、备注
- 按日期倒序排列
- 支持按基金和日期范围筛选

### 交易与持仓页（Tab 结构）

```
┌────────────────────────────────────────┐
│ [当前持仓]  [买入/卖出]  [操作日志]       │
├────────────────────────────────────────┤
│ Tab 内容区域                             │
└────────────────────────────────────────┘
```

- Tab 1「当前持仓」：PositionTable 组件
- Tab 2「买入/卖出」：TransactionForm + 最近交易记录
- Tab 3「操作日志」：TradeLog 组件

### ChatDrawer（AI 分析抽屉）

**触发方式**：
- Dashboard 右下角 AiFab 悬浮按钮
- 各页面 Header 右侧 AI 图标按钮
- 快捷键 `Cmd+K` 全局唤起

**布局**：
- 右侧 Drawer，宽度 400px
- 顶部：标题「AI 分析」+ 关闭按钮
- 中部：消息列表（用户消息右对齐，AI 消息左对齐 + Markdown 渲染）
- 底部：输入框 + 发送按钮
- 输入框上方：快捷建议 Chip（如「分析我的仓位风险」「最近市场怎么样」）

**交互**：
- 发送后显示 typing indicator（三点动画）
- AI 回复支持 Markdown 渲染（加粗、列表、表格）
- SSE 流式输出，逐字显示
- Drawer 关闭时断开 SSE 连接
- 对话历史存储在 Zustand store，页面刷新清空

**AnalysisReport**
- 风险概览卡片：集中度评分、分散度评分、回撤风险等级
- 调仓建议：Markdown 渲染，支持加粗/列表
- 进度步骤条：fetch_data → risk_analysis → advice → complete
